# object-fit內容適應其容器尺寸/圖片相關

object-fit是css中用於控制替換元素(img video)內容適應其容器尺寸的屬性

值：

fill: 預設值 完全填充容器，不抱持寬高比，圖片可能變形

contain:保持內容寬高比，縮放到完全適應容器，容器可能會有空白

cover:保持內容寬高比，縮放到覆蓋容器（超出容器部分可能被裁切），內容無空白

## 關於圖片自身容器高度

---

### 1. 問題的核心：基線對齊 (Baseline)

在 CSS 中，`<img>` 預設是一個 **inline-block**（行內區塊）元素。它的對齊方式預設是 `vertical-align: baseline`。

- **原理：** 圖片會像字母一樣排列在「文字基線」上。
- **空隙來源：** 為了留位置給像 `g`、`j`、`p`、`y` 這種帶有「下延部 (Descenders)」的英文字母，瀏覽器會在基線下方預留一點點空間。即便你的 `<div>` 裡沒有文字，這個空間依然會被撐開。

---

### 2. 解決方法（三選一）

這三種方法都能完美解決空隙問題，你可以根據排版需求選擇：

### 方法 A：改變對齊方式 (最推薦)

將圖片的垂直對齊改為 `top`、`middle` 或 `bottom`。只要不是 `baseline`，空隙就會消失。

CSS

# 

`img {
  vertical-align: middle; /* 或者 top, bottom */
}`

### 方法 B：將圖片轉為區塊元素

將圖片從「行內元素」改為「區塊元素」。區塊元素不遵循文字基線規則，所以不會有空隙。

CSS

# 

`img {
  display: block;
}`

### 方法 C：修改父層的字體大小

既然空隙是為了留給字母用的，把父層的字體大小設為 0，那個空間就會縮到看不見（但這會影響 `p` 標籤，所以你必須在 `p` 裡面重新設定字體大小）。

CSS

# 

`div {
  font-size: 0;
}
p {
  font-size: 16px; /* 需手動設回來 */
}`

---

### 3. 補充：為什麼 `p` 標籤也會有影響？

如果你已經確認 `margin` 是 0，但 `p` 與 `img` 之間還是有距離，請檢查：

- **HTML 的換行：** 如果你的 HTML 程式碼中 `<img>` 和 `<p>` 之間有換行，瀏覽器會將其解析為一個「空格」。
    - **解決：** 將 HTML 寫在同一行，或使用 `display: block` 給圖片。

### 總結建議

通常只要在 CSS 加入 **`img { display: block; }`** 或 **`img { vertical-align: middle; }`** 就能解決 99% 的圖片空隙問題。
