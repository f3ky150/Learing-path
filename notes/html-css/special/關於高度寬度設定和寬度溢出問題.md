# 關於寬度計算

### 1. 塊級元素（Block-level）的「寬度本能」

你不需要給 `html`, `body` 或 `div` 設定 `width: 100%`，它們也會橫向填滿，是因為塊級元素有一種本能叫做 **「自動填充（Fill-available）」**。

- **寬度的預設值是 `auto`**：對於塊級元素，`width: auto` 的意思是「請填滿父元素剩餘的所有水平空間」。
- **即使父元素沒定義寬度**：瀏覽器視窗（Viewport）天生就擁有確定的寬度。`html` 作為塊級元素，會自動撐滿視窗寬度；`body` 再撐滿 `html`；你的 `div` 再撐滿 `body`。
- **這條鏈條是自動連通的**，不需要你手動去寫 `100%`。

### 2.什麼時候「寬度 100%」反而有害？

**這是一個新手常犯的錯誤。如果你給一個 `div` 同時設定了 `width: 100%` 和 `padding`，會發生什麼？**

- **沒寫 `width: 100%` 時**：`width: auto` 會自動扣除 `padding`，讓 `div` 剛好塞進父元素，不會溢出。
- **寫了 `width: 100%` 時**：瀏覽器會強制寬度等於父元素，**再加上**你的 `padding`。結果就是你的 `div` 會超出螢幕邊界，產生討厭的水平滾動條。

> 使用border-size-border-box可以解決這項問題
> 

### 3. 寬度的計算路徑：由外而內 (Top-Down)

寬度的計算就像一條**由上而下的政令傳達**，每一層都知道自己的邊界。

1. **視窗 (Viewport)**：這是源頭，它告訴 `<html>`：「我的寬度是 1440px」。
2. **`<html>` 元素**：它是 `width: auto`，所以它接收了 1440px，並告訴 `<body>`：「我現在是 1440px」。
3. **`<body>` 元素**：同樣是 `width: auto`，它接收了 1440px，並告訴你的 `.parent` 盒子：「我的空間是 1440px」。
4. **`.parent` 盒子**：即便是 `width: auto`，它也已經從 `<body>` 那裡繼承了「最大可用空間」。
5. **你的子級 (Child)**：當你設定 `width: 100%` 時，它會回頭問 `.parent`：「你的可用空間是多少？」這時 `.parent` 已經有一個從祖先那裡傳下來的確定數值了。

- **寬度 (Width)**：預設遵循 **Available Space（可用空間）**。父級就算沒定義寬度，它也知道自己「最大能用多少」。
- **高度 (Height)**：預設遵循 **Content Size（內容尺寸）**。父級如果沒定義高度，它就認為自己「沒有預留空間」。

寬度用「由外向內」的百分比或彈性佈局：控制它在不同螢幕下的佔比。

高度用「由內向外」的自動撐開：確保文字換行或內容增減時，排版不會壞掉。

屬性類別 設定方向 常用單位
寬度 (Width) 從外向內 %, vw, max-width, calc()
高度 (Height) 從內向外 auto (靠內容撐開), padding
