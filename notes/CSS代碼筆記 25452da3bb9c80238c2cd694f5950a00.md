## CSS特性

## 繼承性

子級會繼承父級的**文字控制屬性**

原本元素中自身有樣式的話 不會繼承擁有的樣式 ex h1當中的文字大小不繼承 a當中的顏色+底線樣式不繼承

### **以下是幾類常見的、不會被子元素繼承的 CSS 屬性：**

- **盒模型屬性 (Box Model Properties)**：
    - `width`、`height`
    - `margin`
    - `border`
    - `padding`
    - 這些屬性定義了元素本身的尺寸、邊框和間距，因此不會被子元素繼承。舉例來說，你不會希望一個子元素的內邊距會跟著父元素一起變動，因為它們是兩個獨立的個體。
- **定位屬性 (Positioning Properties)**：
    - `top`、`right`、`bottom`、`left`
    - `position`
    - `z-index`
    - 這些屬性控制元素在頁面上的位置，是針對單個元素的，自然不會繼承。
- **背景屬性 (Background Properties)**：
    - `background-color`
    - `background-image`
    - `background-position`
    - `background-repeat`
    - `background-size`
    - 除非你特別設定，否則子元素的背景不會自動和父元素一樣。
- **其他常見屬性**：
    - `display`
    - `overflow`
    - `float`
    - `clear`
    - `vertical-align`
    - `cursor`

### 會繼承的屬性

相較之下，會被繼承的屬性主要與**文本和排版**有關，例如：

- **字體相關 (Font)**：`font-family`、`font-size`、`font-style`、`font-weight` 等。
- **顏色與文本 (Color & Text)**：`color`、`text-align`、`line-height`、`text-indent`、`letter-spacing`、`word-spacing` 等。
- **列表樣式 (List Style)**：`list-style-type`、`list-style-image`、`list-style-position`。
- **可見性 (Visibility)**：`visibility`。

## 覆蓋性

不同的屬性會疊加 ex div{font-size:30px} div{font-weight:700} 同時生效

## 優先級

當一個標籤使用多種選擇器，基於不同種類選擇器優先順序

公式:通配符選擇器(ex:body{})<標籤選擇器(Ex:div{} p{} span)<類選擇器(ex: class{})<id選擇器<行內樣式<!important

選中範圍越大 優先度越小

有多個選擇器(子/後代) 依照上述公式的個數決定權重 

比較重要的是 如果有繼承 權重最低  EX. #div但控制的是div裡的子級 子級繼承div的控制屬性

### ***關於優先級的延伸知識***

CSS 選擇器的**優先級（Specificity）和維護性**。

```html
 <div class="banner">

        <div class="wrapper">

            <div class="left">left</div>

            <div class="right">right</div>

        </div>

    </div>
```

在這段 HTML 程式碼中，如果要選擇left這個類別  `.banner .left` **是更好的選擇。

以下是詳細的解釋：

### 為什麼 `.banner .left` 比較好？

1. **選擇器越短越好**：在 CSS 規範中，選擇器越短，效能通常越好，雖然在現代瀏覽器中這個差異已經微乎其微。但更重要的是，簡短的選擇器能夠提升**可讀性**和**維護性**。
2. **避免過度指定（Over-specificity）**：`.banner .wrapper .left` 的選擇器優先級（Specificity）比 `.banner .left` 高。這代表它有更強的「權重」，如果未來你需要修改 `.left` 的樣式，使用 `.banner .wrapper .left` 會讓你的 CSS 難以被其他更通用的選擇器覆蓋。這會導致你需要寫更多複雜的 CSS 規則來覆蓋之前的設定，使程式碼變得混亂。
3. **遵循 CSS 最佳實踐**：**BEM（Block Element Modifier）** 等主流 CSS 方法論都提倡**降低選擇器的優先級**。它們主張使用單一的類別名稱（例如 `.banner__left`），而不是透過多層級的後代選擇器來指定樣式。這樣做可以讓你的 CSS 更模組化、可重複使用。

### 何時會需要使用多層後代選擇器？

雖然通常不推薦，但多層後代選擇器在某些特定情況下還是有其用途，例如：

- **避免樣式衝突**：當你使用的第三方 CSS 庫，有一個 `.left` 的類別名稱，並且它的樣式與你的設計衝突。這時你可能會需要使用 `.banner .wrapper .left` 來確保你的樣式能夠覆蓋它。
- **針對特定情境**：你希望在特定父元素（例如 `.dark-theme`）下的 `.wrapper` 裡的 `.left` 有不同的樣式。

```css
.dark-theme .wrapper .left {
  color: white;
}
```

結論:
在這個例子中，`.banner .wrapper .left` 雖然可以正確選取到元素，但它多指定了 `.wrapper` 這個不需要的父層，增加了不必要的**權重**和**複雜性**。
